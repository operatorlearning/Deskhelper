# Desktop AI Agent 项目技术综述报告

## 1. 项目概览
**Desktop AI Agent** 是一款基于大模型驱动的下一代桌面智能化辅助工具。它通过结合多模态视觉理解（Qwen2.5-VL）、语音交互（Whisper/TTS）和自动化控制技术，实现了“理解屏幕 -> 规划任务 -> 自动执行”的闭环体验。

## 2. 核心技术栈
- **视觉模型**：Qwen2.5-VL（集成于 Ollama），支持高精度的屏幕截图分析与 OCR 识别。
- **语音系统**：
  - **STT (语音转文字)**：OpenAI Whisper (Medium)，实现毫秒级响应的语音指令识别。
  - **TTS (文字转语音)**：pyttsx3 / edge-tts，赋予 Agent 拟人化的反馈能力。
- **大脑引擎**：LangChain 框架，用于处理复杂的逻辑推理与长短期记忆（ChromaDB）。
- **自动化控制**：PyAutoGUI + PyWinAuto，实现跨应用的点击、输入及系统级操作。
- **前端架构**：Gradio 4.40+，并经过深度定制化的 CSS 重构。

## 3. 本次重点优化内容

### 3.1 视觉与交互层（UI/UX）
- **现代化设计**：引入 GitHub Dark 主题与毛玻璃（Glassmorphism）特效，重塑了对话气泡与面板质感。
- **桌面原生化**：
  - 开发了 `run_desktop.py` 启动器，利用 `pywebview` 技术将 Web 页面转化为**独立桌面小应用**。
  - 支持 **“始终置顶”** 浮窗模式，适配 460x720 紧凑型布局。
  - 移除了自动弹出浏览器的逻辑，使体验更接近原生软件。

### 3.2 环境兼容性修复
- **依赖冲突解决**：强制降级 NumPy 至 `1.26.4` 以修复与 pandas 等库的二进制兼容错误（NumPy 2.x 冲突）。
- **编码优化**：解决了 Windows CMD 环境下读取 `requirements.txt` 时的 GBK 编码报错问题。
- **稳定性增强**：修复了 UI 回调中 `analysis` 变量未定义的逻辑缺陷。

### 3.3 架构健壮性
- **进程管理**：优化了后端服务启动逻辑，支持 Windows 平台下的进程优雅退出。
- **端口策略**：将默认 UI 端口迁移至 `7865`，并提供了残留进程自动清理建议，解决了因端口占用导致无法启动的问题。

## 4. 功能模块说明
- **💬 智能对话**：支持文字、图片、语音混合输入，具备上下文记忆。
- **🖥 屏幕分析**：一键捕获当前窗口并由视觉模型进行深度解析。
- **⚡ 自动化任务**：根据用户自然语言指令，自动规划并执行桌面操作流。
- **🧠 记忆管理**：基于 ChromaDB 的长期记忆存储，Agent 会随着使用时间的增长越来越懂你。

## 5. 快速启动指南
1. 确保后台运行 `ollama serve` 并加载相关模型。
2. 终端执行：`python run_desktop.py`。
3. 通过桌面浮窗开启您的 AI 代理之旅。
